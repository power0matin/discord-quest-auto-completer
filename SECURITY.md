# Security Policy

## Supported Versions
This project is provided “as-is” and is not guaranteed to be maintained on a fixed schedule.  
Security fixes (if any) are typically applied to the latest `main` branch and the most recent release.

| Version | Supported |
|--------:|:---------:|
| Latest release | ✅ |
| `main` branch  | ✅ |
| Older tags/releases | ❌ |

## Reporting a Vulnerability
If you believe you have found a security vulnerability, please report it **privately** and **responsibly**.

### Preferred Contact
- Email: **me@matinshahabadi.ir**
- Subject line: **[SECURITY] discord-quest-auto-completer — <short summary>**

### What to Include
To help reproduce and assess the impact, please include:
- A clear description of the issue and potential impact
- Steps to reproduce (minimal PoC, if applicable)
- Affected environment (OS, Discord client version, and any relevant settings)
- Relevant logs, screenshots, or stack traces (redact sensitive data)
- Whether the issue is reliably reproducible

### Please Do Not
- Do **not** open public GitHub issues for security reports
- Do **not** publish exploit code, bypass techniques, or step-by-step abuse instructions
- Do **not** include secrets/tokens/cookies or personal data in your report

## Response and Disclosure Timeline
I will try to acknowledge valid reports within **7 days**.
If confirmed, I will work on a fix and coordinate a responsible disclosure timeline where possible.  
Timelines may vary depending on severity and availability.

## Scope
Security reports may include (but are not limited to):
- Supply-chain risks, malicious dependencies, or suspicious scripts
- Unsafe defaults, credential leakage, or sensitive data exposure
- Remote code execution, privilege escalation, or command injection
- Any behavior that could enable abuse beyond the documented intent

## Safe Harbor
Good-faith security research and responsible disclosure are appreciated.  
Please avoid actions that could harm users, violate privacy, or disrupt services.
